Eventos, target y capturar eventos desde el teclado:

Eventos:

En JavaScript, un evento es una acción o suceso que ocurre en un elemento del DOM (Document Object Model) y que puede ser detectado y controlado mediante código. Los eventos pueden ser generados por el usuario, como hacer clic en un botón, mover el mouse, escribir en un campo de texto, o pueden ser generados automáticamente por el navegador, como cargar la página o cambiar el tamaño de la ventana.
Cuando ocurre un evento en un elemento, JavaScript puede ejecutar una función específica asociada a ese evento. Estas funciones se conocen como "manejadores de eventos" o "event listeners". Los event listeners se encargan de escuchar y responder a los eventos ocurridos en un elemento específico.
El uso de eventos en JavaScript permite interactuar con el usuario, capturar acciones y comportamientos, y ejecutar acciones personalizadas en respuesta a estos eventos siendo éstos fundamentales en la programación web para crear interactividad y dinamismo en las aplicaciones y sitios web.

    Los eventos a escuchar son:
  • blur: Cuando el elemento pierde el foco.
  • click: El usuario hace clic sobre el elemento.
  • dblclick: El usuario hace doble clic sobre el elemento.
  • focus: El elemento gana el foco.
  • keydown: El usuario presiona una tecla.
  • keypress: El usuario presiona una tecla y la mantiene pulsada.
  • keyup: El usuario libera la tecla.
  • load: El documento termina su carga.
  • mousedown: El usuario presiona el botón del ratón en un elemento.
  • mousemove: El usuario mueve el puntero del ratón sobre un elemento.
  • mouseout: El usuario mueve el puntero fuera de un elemento.
  • mouseover: El usuario mantiene el puntero sobre un elemento.
  • mouseup: El usuario libera el botón pulsado del ratón sobre un elemento.
  • unload: El documento se descarga, bien porque se cierra la ventana, bien porque se navega a otra página.


Esto no significa que se utilicen todos ellos pero es importante saber que existen y que están como recursos por si precisamos generar acciones sobre estas interacciones del usuario. Para conocer más sobre este tema y ver ejemplos prácticos te invitamos a ver la siguiente presentación.

Target:

En JavaScript, el objeto event.target se refiere al elemento en el cual se originó un evento. Cuando se utiliza un event listener, el valor de event.target es el elemento HTML específico que disparó el evento.
Por ejemplo, si tienes un event listener en un botón y haces clic en ese botón, event.target será una referencia al propio botón. Esto te permite acceder y manipular las propiedades y atributos del elemento específico que activó el evento. Eventos de teclado:

En JavaScript, se pueden capturar eventos del teclado utilizando los event listeners adecuados. Algunos de los eventos del teclado más comunes son:

  • keydown: Se dispara cuando una tecla se presiona y se mantiene presionada.
  • keyup: Se dispara cuando se suelta una tecla después de haberla presionado.
  • keypress: Se dispara cuando se presiona y se suelta una tecla.

Pueden ser interacciones sencillas pero sumamente útiles para funcionalidad del sitio web. Te mencionamos algunos ejemplos donde se pueden aprovechar:

  • Validación de formulario: Puedes utilizar estos eventos para realizar validaciones en tiempo real en los campos de formulario mientras el usuario escribe. Por ejemplo, puedes verificar si un campo de código postal es válido después de que el usuario presione una tecla (keydown) o cuando la tecla se suelte (keyup).
  • Búsqueda de productos: Puedes utilizar el evento keydown para capturar cuando el usuario presiona la tecla "Enter" después de ingresar un término de búsqueda. De esta manera, puedes realizar la búsqueda de productos en el sitio web y mostrar los resultados correspondientes.
  • Filtros de productos: Si tienes filtros interactivos en tu sitio web de comercio electrónico, como filtros por precio, talla o color, puedes utilizar los eventos keydown, keyup o keypress para detectar cuando el usuario interactúa con esos filtros y aplicar los cambios en tiempo real.
  • Navegación por teclado: Puedes utilizar estos eventos para mejorar la accesibilidad de tu sitio web de comercio electrónico, permitiendo a los usuarios navegar por los productos y las páginas utilizando las teclas de flecha o atajos de teclado específicos.
  • Carrito de compras: Puedes utilizar estos eventos para permitir al usuario agregar o eliminar elementos del carrito de compras utilizando atajos de teclado. Por ejemplo, al presionar una combinación de teclas, se puede agregar un producto específico al carrito sin tener que hacer clic en el botón correspondiente.

La elección de qué evento utilizar y cómo aprovecharlo dependerá de las necesidades específicas de tu sitio web y la interacción que deseas lograr con los usuarios.
